# Event Reflection “Saturday Meet up”

### Purpose of the Event

- Share knowledge and practical experience across diverse tech fields: Cloud, Cyber Security, Game Networking, AI/GenAI, and Containerization[cite: 4, 5, 6, 7, 8, 9].
- Provide a practical perspective on the career development journey from foundational roles (Helpdesk/Sysadmin) to Senior and Cloud/DevOps levels[cite: 8].
- Cultivate essential soft skills such as teamwork and an IT infrastructure operational mindset[cite: 4, 8].

### List of Speakers

- **Truong Huy Phuoc** - Presented on The Art of Effective Teamwork[cite: 4].
- **Nguyen Quoc Bao** - Presented on Multiplayer in the Cloud (Godot + AWS WebSockets)[cite: 5].
- **Le Hoang Gia Dai** - Presented on Machine Learning-based Network Intrusion Detection System (NIDS) on AWS[cite: 6].
- **Bao Huynh** - Junior Cloud Native Developer at Endava Vietnam, presented on Docker & Containerization[cite: 7].
- **Tran Trung Vinh** - System Administrator at Central Retail Group, shared the journey from IT Helpdesk to Senior Sysadmin[cite: 8].
- **Viet Phat** - Presented on building GraphRAG applications using Amazon Bedrock and Amazon Neptune[cite: 9].

### Key Highlights

#### The Art of Effective Teamwork

- **4 Golden Rules:** Clear & Shared Goals, Right Person - Right Place, Open Communication & Active Listening, and Personal Accountability[cite: 4].
- The importance of digital tools (Trello, ClickUp, Google Workspace, Slack, Discord) in optimizing and synchronizing team workflows[cite: 4].

#### Multiplayer in the Cloud with Godot & AWS WebSockets

- Guide to building a Serverless architecture for multiplayer games using **AWS API Gateway, AWS Lambda, and Amazon DynamoDB**[cite: 5].
- Comparison of WebSocket advantages (suitable for turn-based/lobby games) over UDP or HTTP Polling[cite: 5].
- How the Godot Client integrates `WebSocketPeer` to send and receive messages via WebSockets, handle matchmaking, and update real-time game states[cite: 5].

#### Security with AWS WAF & Machine Learning NIDS

- Analysis of the limitations of traditional rule-based (Signature-based) firewalls against Zero-day attacks or anomalous behaviors[cite: 6].
- Combining **AWS WAF** with a **Machine Learning (NIDS)** model to enhance proactive intrusion detection capabilities based on real network data (using the CSE-CIC-IDS2018 dataset)[cite: 6].

#### Containerization with Docker

- Distinguishing between Virtualization (traditional VMs) and Containerization[cite: 7].
- Containers are more lightweight, share the host OS, boot in milliseconds, and consume fewer resources[cite: 7].
- Basic structure: Dockerfile, Images, and Containers[cite: 7].
- High applicability in CI/CD pipelines, Microservices architectures, and Cloud-native applications[cite: 7].

#### Career Journey from IT Helpdesk to Senior Sysadmin

- Practical lesson: Don't just support systems, understand how they are built[cite: 8].
- Master Linux and Networking fundamentals, and build hands-on lab environments[cite: 8].
- **Operational Mindset:** Always prevent first before fixing (Prevent first - fix later), automate repetitive tasks, and follow the ultimate rule: "Never test in production"[cite: 8].

#### GenAI: Building GraphRAG Applications

- Traditional RAG often struggles with reasoning complex relationships[cite: 9].
- GraphRAG solves this with "Multi-hop Reasoning" capabilities[cite: 9].
- Introduction to an integrated architecture between **Amazon Bedrock** (LLM, Embeddings) and **Amazon Neptune** (Graph Database) to store nodes/edges and query graph-based knowledge[cite: 9].

### What I Learned

#### Technical & Architecture Thinking

- Clearly understand the shift from monolithic On-Premise infrastructure to Cloud, and from Virtual Machines (VMs) to modern Container (Docker) / Serverless architectures[cite: 5, 7, 8].
- Recognize the role of AI/ML not just in chatbots but also in automating security (ML-based Security) and improving the accuracy of information retrieval systems (GraphRAG)[cite: 6, 9].

#### Soft Skills & Career Orientation

- Practical experience and personal project portfolios are much more valuable than just collecting certificates[cite: 8].
- Go deep into 1-2 core skills before expanding[cite: 8].
- Communication and teamwork skills are key to project success[cite: 4].
- Applying the 4 golden rules correctly will help the team coordinate smoothly and minimize conflicts[cite: 4].

### Practical Application

- **Use Docker:** Start containerizing small personal apps/projects using Dockerfiles for consistent dev environments and easy deployment[cite: 7].
- **Upgrade Project Security:** Learn more about WAF or apply NIDS concepts to monitor logs and block malicious requests for web applications under development[cite: 6].
- **Practice Cloud & Game Dev:** Try building a simple client-server model using WebSockets (or connecting APIs via n8n) as a hands-on exercise to get familiar with Serverless[cite: 5].
- **Improve Teamwork:** Clearly set goals and systematically establish rules for using shared management tools (Trello/Slack) for group assignments at school[cite: 4].
